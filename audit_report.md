# Daemon.md Security & Performance Audit Report

## 🚨 Security Findings

### 1. Critical: Directory Traversal Vulnerability (Path Injection)
**Location:** `daemon.py`
**Description:** The ingestion engine blindly trusts the `filepath` output provided by the Gemini LLM. It constructs the target path for writing file updates using: `target_path = VAULT_DIR / rel_path`.
If a malicious actor (or even hallucinated LLM output) constructs a `rel_path` such as `../../../../../etc/passwd` or `../../.ssh/authorized_keys`, the daemon will happily overwrite those files with the LLM's generated content, leading to arbitrary file overwrite.
**Recommendation:** Implement strict path normalization and validation. Ensure that the resolved `target_path` is strictly a child directory of `VAULT_DIR` using `target_path.resolve().is_relative_to(VAULT_DIR.resolve())`.

### 2. High: AppleScript Command Injection
**Location:** `daemon.py` and `lint_wiki.py` (`send_notification` function)
**Description:** The system sends native macOS notifications by constructing an AppleScript string dynamically and executing it via `subprocess.run(["osascript", "-e", apple_script])`. Even though backslashes and double quotes are escaped, this approach of interpolating strings into executable code is inherently risky and can be bypassed.
**Recommendation:** Pass variables securely as positional arguments to the script, rather than interpolating them directly into the script string. For example: `osascript -e 'on run argv' -e 'display notification (item 2 of argv) with title (item 1 of argv)' -e 'end run' "$title" "$message"`.

### 3. Low: Environment File Secrets in Source Control (Potential)
**Location:** General architecture
**Description:** While `.env.example` is used and `.env` is in `.gitignore`, the `lint_wiki.py` script attempts to import `APIRedactingFormatter` but misses importing `sys` properly at the top, leaving it scattered. The `daemon.py` redacts logs perfectly, but standard Python exceptions generated by modules might accidentally leak strings.

## ⚡ Performance Findings

### 1. Critical: Blocking Watchdog Event Dispatcher Thread
**Location:** `daemon.py`
**Description:** `RawFolderHandler` responds to `on_created` and `on_modified` events by calling `safe_process_raw_file(event.src_path)`, which subsequently triggers synchronous calls to the Gemini API via `process_raw_file`. The Gemini API call can take several seconds to complete. Because `watchdog` processes events sequentially in its dispatcher thread, blocking it will cause the system to queue up and delay processing for all subsequent filesystem events. If a burst of files arrives, the queue may drop events or significantly drift behind real-time.
**Recommendation:** Offload the `safe_process_raw_file` execution to a background worker queue or utilize the `concurrent.futures.ThreadPoolExecutor` to handle processing without blocking the `watchdog` dispatcher.

### 2. High: Unnecessary File System Polling Wait Time
**Location:** `daemon.py` (Main Loop)
**Description:** The fallback `periodic_scan()` executes within a `while True` block containing `time.sleep(60)`. If a file is skipped or watchdog fails, the user must wait up to a minute for the fallback to pick it up. In a locally run service, this polling interval can be reduced or dynamically backed off depending on queue state, especially given the `processing_files` lock.
**Recommendation:** Decouple the fallback loop's sleep interval from the main thread blocking, or make the interval configurable and shorter.

### 3. Medium: Excessive Token Usage on Full Wiki Reads
**Location:** `lint_wiki.py`
**Description:** `collect_wiki_contents()` loads the raw text of *every single markdown file* in the `wiki/` directory and includes it verbatim in the prompt. While the reasoning model handles large contexts, an actively growing vault will rapidly push the context limit and result in extremely high token consumption on every run.
**Recommendation:** While this may be by design for the "Synthesis Linter", consider applying a chunking or hierarchical map-reduce strategy for the audit if the total token length exceeds a safe threshold (e.g., 80% of the model's max context window).

### 4. Low: Repeated Directory Checks in `build_graph()`
**Location:** `graph_builder.py`
**Description:** `build_graph()` executes an `os.walk(wiki_dir)` twice—once to catalog all IDs, and a second time to read the contents. This implies iterating over the filesystem and potentially hitting disk limits for large repositories.
**Recommendation:** Refactor to execute a single pass over the directory structure, saving memory and disk I/O operations.
